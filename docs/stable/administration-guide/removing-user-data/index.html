Removing user data

Removing user data according to GDPR

The General Data Protection Regulation (GDPR) law enforces the right for individuals to have personal data erased.

The following procedure describes how to remove the Che user’s data.

The following commands use the default Kubernetes namespace or OpenShift project, eclipse-che, as a user’s example for the -n option.
Prerequisites

  • A user or an administrator authorization token. To delete the Che data bound to a user account, admin privileges are required. The admin is a special Che administrator account that should be pre-created and enabled using the CHE_SYSTEM_ADMIN__NAME and CHE_SYSTEM_SUPER__PRIVILEGED__MODE = true properties of the Che Custom Resource.

    spec:
 server:
   customCheProperties:
     CHE_SYSTEM_SUPER__PRIVILEGED__MODE: 'true'
     CHE_SYSTEM_ADMIN__NAME: '<admin-name>'

    If needed, use kubectl commands below for patching the Che Custom Resource:

    $ kubectl patch checluster/eclipse-che \
  --type merge \
  -p '{ "spec": { "server": {"customCheProperties": {"CHE_SYSTEM_SUPER__PRIVILEGED__MODE": "true"} } }}' \
  -n eclipse-che
    $ kubectl patch checluster/eclipse-che \
  --type merge \
  -p '{ "spec": { "server": {"customCheProperties": {"CHE_SYSTEM_ADMIN__NAME": "<admin-name>"} } }}' \
  -n eclipse-che

    All Che system permissions are granted to the administrative user. To configure the administrative user, use the CHE_SYSTEM_ADMIN__NAME property. The default value is admin. The system permissions are granted when the Che server starts. If the user record is not in the Che user database, the permissions are granted after the first login of the user.

    Authorization token privileges:

    • admin - Can delete all personal data of all users

    • user - Can delete only the data related to the user

  • A user or an administrator is logged in the OpenShift cluster with deployed Che.

  • A user ID is obtained. Get the user ID using the commands below:

    • For the current user:

      $ curl -X GET \
  --header 'Authorization: Bearer <user-token>' \
  'https://<che-host>/api/user'

    • To find a user by name:

      $ curl -X GET \
  --header 'Authorization: Bearer <user-token>' \
  'https://<che-host>/api/user/find?name=<username>'
Example of obtaining a user ID

This example uses vparfono as a local user name.

$ curl -X GET \
  --header 'Authorization: Bearer <user-token>' \
  'https://che-vp-che.apps.che-dev.x6e0.p1.openshiftapps.com/api/user/find?name=vparfono'

The user ID is at the bottom of the curl command output.

{
 "name": "vparfono",
 "links": [
   {
.
.
.
   }
 ],
 "email": "vparfono@redhat.com",
 "id": "921b6f33-2657-407e-93a6-fb14cf2329ce"
}
Procedure

  • Remove the data using the API:

    $ curl -i -X DELETE \
  --header 'Authorization: Bearer <user-token>' \
  https://<che-host>/api/user/<user-id>

The procedure can be also executed using the Swagger UI available on https://<che-host>/swagger
Verification

Running the following command returns code 204 as the API response:

$ curl -i -X DELETE \
  --header 'Authorization: Bearer <user-token>' \
  https://<che-host>/api/user/<user-id>

The following procedure only removes the user on the Che level. For removing all Kubernetes user data including namespaces from the cluster, the Kubernetes cluster-admin needs to explicitly remove the Kubernetes resources bound to the user.
Additional resources

To remove the data of all users, follow the instructions for Uninstalling Che.
